KWIK WINDOWS BUILD GUIDE:

get vs2017 (it's what I use, newer vers should work tho)
get the c++ compiler & cmake

open x64 Native Tools Command Prompt for VS 2017 (or wtv ver ur using) as admin

download glfw https://www.glfw.org/ & extract

cd to/where/u/extracted/glfw-3.4 && mkdir build && cd build

cmake -S .. -B .. -A x64 -DCMAKE_INSTALL_PREFIX=C:\Libraries\glfw
cmake --build . --config Release
msbuild INSTALL.vcxproj /p:Configuration=Release

download bzip2 & extract (next to glfw-3.4 folder)
https://github.com/libarchive/bzip2

cd to/where/u/extracted/bzip2-master && mkdir build && cd build

cmake -S .. -B .. -A x64 -DCMAKE_INSTALL_PREFIX=C:\Libraries\bzip2
cmake --build . --config Release (you may get an error but as long as bzip2.lib gets built ur good)
msbuild INSTALL.vcxproj /p:Configuration=Release (you'll also get an error here but it'll still copy)

download zlib & extract 
https://github.com/madler/zlib/releases/tag/v1.3.2

cd to/where/u/extracted/zlib-1.3.2 && mkdir build && cd build

cmake -S .. -B .. -A x64 -DCMAKE_INSTALL_PREFIX=C:\Libraries\zlib
cmake --build . --config Release
msbuild INSTALL.vcxproj /p:Configuration=Release

I've also just included my Libraries folder for easy use, so you can just extract it to C:\ but I do recommend learning how to compile these libs yourself!

clone kwik (maybe next to the glfw-3.4 folder)

cd ../../kwik/build

cmake .. -A x64 -Dglfw3_DIR=C:\Libraries\glfw\lib\cmake\glfw3 -DZ_LIB=C:/Libraries/zlib/lib/zlib.lib -DBZ2_LIB=C:/Libraries/bzip2/lib/bz2.lib ..
cmake --build . --parallel

and then copy bz2-1.dll, z.dll to build\compiler\Debug (or release)

after kwikc finishes you can build your game with:

cd [exportfoldername] && mkdir build && cd build

cmake -S .. -B .. -A x64 -Dglfw3_DIR=C:\Libraries\glfw\lib\cmake\glfw3
cmake --build . --config Debug --parallel
